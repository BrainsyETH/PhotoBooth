# Screenshot manifest

These files are **placeholders** — solid parchment PNGs at the correct
dimensions so the site builds without 404s. Replace each one with a real
product screenshot/render at (at least) the dimensions below. Keep the same
filename and aspect ratio so the layout stays correct. The marketing pages
also render a visible "REPLACE: …" label over each slot until you swap in the
real art; that label disappears automatically once the real image fills the
frame (the placeholders are flat parchment, the real screenshots will not be).

| File              | Dimensions (px) | Aspect | What it should show |
|-------------------|-----------------|--------|---------------------|
| `hero-tablet.png` | 1200 × 1500     | 4:5    | The booth running on a tablet, ideally framed as a tablet on a stand (the hero device mockup). Show the live capture/attract screen so a first-time visitor instantly "gets" the product. |
| `attract.png`     | 1080 × 1620     | 2:3    | The **attract / idle** screen guests see before tapping — event name, tagline, "tap to start". |
| `capture.png`     | 1080 × 1620     | 2:3    | The **capture** screen mid-session — countdown or the single/collage/GIF chooser. |
| `share.png`       | 1080 × 1620     | 2:3    | The **share** screen — email entry + QR tile, where the guest gets their photo. |

Notes:
- PNG preferred (transparent or parchment background both fine).
- Higher resolution is OK; the browser scales down. Avoid going below the
  listed dimensions or the images will look soft on retina displays.
- Do not crop the device chrome out of `hero-tablet.png` if you shoot a real
  tablet — the framing is part of the "lives on your tablet" story.
